# Style and Conventions
- Frontend is hand-written static markup with a shared stylesheet and one shared script.
- Naming uses kebab-case CSS classes and semantic section naming (`site-header`, `mobile-menu`, `section-title`, etc.).
- JavaScript is wrapped in an IIFE, uses strict mode, DOM query selectors, and concise arrow functions.
- Comments are short section headers that separate behavior blocks in `script.js` and layout blocks in `style.css`.
- No framework conventions, lint config, or formatter config were found; preserve the existing vanilla HTML/CSS/JS style.
