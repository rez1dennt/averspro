# Code Review Skill

Запускать этот чеклист ПЕРЕД тем как сказать пользователю что задача выполнена.
Не пропускать пункты, не говорить «готово» пока не прошёл по всем.

## 1. Задача выполнена?

- [ ] Сделано именно то что просили — не больше, не меньше
- [ ] Все части задачи закрыты (если было несколько требований)
- [ ] Ничего не сломалось из того что работало раньше
- [ ] Крайние случаи обработаны: пустые данные, длинные строки, ноль элементов

## 2. Безопасность

- [ ] Нет вставки пользовательских данных через `innerHTML`
- [ ] Формы валидируются перед отправкой
- [ ] Нет секретов в коде (токены, пароли, ключи)
- [ ] Ссылки `target="_blank"` имеют `rel="noopener noreferrer"`

## 3. Читаемость

- [ ] Имена переменных и функций объясняют намерение, не механику
- [ ] Нет комментариев которые объясняют очевидное
- [ ] Нет мёртвого кода — закомментированных блоков, неиспользуемых переменных
- [ ] Вложенность разумная — не больше 3 уровней без веской причины
- [ ] Новый код вписывается в стиль окружающего кода

## 4. Дублирование

- [ ] Один и тот же блок логики не повторяется более двух раз
- [ ] Если повторяется — вынесен в переменную или функцию
- [ ] Не создана новая функция ради единственного вызова (лишняя абстракция)

## 5. HTML/CSS/JS специфично для этого проекта

- [ ] Новые изображения имеют `loading="lazy"`, `decoding="async"`, `alt`
- [ ] Анимации через `transform`/`opacity`, не через `width`/`height`/`top`
- [ ] Новые стили добавлены в `style.css`, а не в `<style>` тег внутри HTML
- [ ] Новая логика добавлена в `script.js`, а не в `<script>` тег внутри HTML
- [ ] Шапка и футер одинаковы на всех страницах если менялись

## 6. Производительность

- [ ] Нет синхронных операций которые блокируют рендер
- [ ] Нет тяжёлых вычислений в обработчиках `scroll` или `resize` без throttle
- [ ] DOM не читается и не записывается вперемешку в цикле

## 7. Финальная проверка

Честно ответить на три вопроса:
1. Другой разработчик поймёт этот код после одного прочтения?
2. Я бы принял этот код на ревью от коллеги без замечаний?
3. Я ничего не сделал «проще» за счёт того что оно стало слабее или небезопаснее?

Если хотя бы один ответ «нет» — исправить перед тем как говорить «готово».
