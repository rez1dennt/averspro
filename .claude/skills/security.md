# Security Skill

Применять при любом изменении форм, пользовательского ввода, вывода данных на страницу.

## Главное правило

Никогда не доверять данным, которые пришли снаружи: от пользователя, из URL, из localStorage, из API.

## Формы и пользовательский ввод

- Валидировать на клиенте — для UX
- Валидировать на сервере — для безопасности (клиентскую можно обойти)
- Поля с типом `email`, `tel`, `url` — проверять regex дополнительно, не только атрибут `type`
- Обязательные поля — проверять что не пустые и не только пробелы
- Длину строк — ограничивать, чтобы не принять 10 000 символов там где ждёшь 50

## XSS — самое частое на фронтенде

- **Никогда не вставлять пользовательские данные через `innerHTML`**
- Использовать `textContent` вместо `innerHTML` когда вставляешь текст
- Если нужен HTML из данных — санировать через DOMPurify или аналог
- `document.write()` — не использовать вообще
- Атрибуты элементов из данных — устанавливать через `setAttribute`, не через конкатенацию строк в HTML

```js
// ПЛОХО
element.innerHTML = userInput

// ХОРОШО
element.textContent = userInput
```

## Ссылки и URL

- Ссылки с `target="_blank"` — добавлять `rel="noopener noreferrer"`
- Не строить URL через конкатенацию пользовательских данных
- Редиректы по параметру URL — проверять что ведут на свой домен

## Конфиденциальные данные

- Не хранить пароли, токены, ключи API в коде или комментариях
- Не логировать чувствительные данные в `console.log`
- Не класть секреты в localStorage (читается любым скриптом на странице)

## HTTP-заголовки (если есть контроль над сервером)

- `Content-Security-Policy` — ограничить источники скриптов и стилей
- `X-Frame-Options: DENY` — защита от clickjacking
- `X-Content-Type-Options: nosniff` — запрет подмены MIME

## Зависимости

- Не добавлять библиотеки из непроверенных источников
- Подключать внешние скрипты только через `integrity` + `crossorigin` (SRI)
- Проверять что CDN-ссылка не ведёт на произвольный сторонний ресурс

## Чеклист при работе с формой

- [ ] Валидация на клиенте есть
- [ ] Пустые и пробельные значения обработаны
- [ ] Данные не вставляются в DOM через innerHTML
- [ ] Нет console.log с данными пользователя
- [ ] Ссылки `target="_blank"` имеют `rel="noopener noreferrer"`
