# Performance Skill

Применять когда добавляю изображения, медиа, скрипты, стили, анимации, или работаю с DOM.

## Изображения — самое частое узкое место

- Всегда добавлять `loading="lazy"` на изображения вне первого экрана
- Изображения первого экрана (hero, логотип) — `loading="eager"` или без атрибута
- Всегда добавлять `decoding="async"` — позволяет браузеру декодировать в фоне
- Всегда указывать `width` и `height` — браузер резервирует место, нет сдвига layout (CLS)
- Формат: предпочитать `.webp` над `.jpg`/`.png` — в среднем на 30% легче
- Не растягивать маленькое изображение CSS — использовать изображение нужного размера
- Для декоративных изображений без смысла — `alt=""`, для контентных — осмысленный alt

```html
<!-- ПЛОХО -->
<img src="photo.jpg">

<!-- ХОРОШО -->
<img src="photo.webp" alt="Описание" width="800" height="450" loading="lazy" decoding="async">
```

## Блокировка рендера — скрипты и стили

- Скрипты в конце `<body>` или с атрибутом `defer` — не блокируют парсинг HTML
- `defer` — скрипт загружается параллельно, выполняется после парсинга DOM, по порядку
- `async` — только для независимых скриптов (аналитика, виджеты), порядок не гарантирован
- Критические стили (выше линии сгиба) — инлайнить в `<head>` если страница медленная
- Внешние шрифты — добавлять `<link rel="preconnect">` до ссылки на шрифт

```html
<!-- Хороший порядок в <head> -->
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="stylesheet" href="style.css">

<!-- Скрипты в конце body -->
<script src="script.js" defer></script>
```

## Перекраски DOM — анимации и изменения стилей

- Анимировать только `transform` и `opacity` — они не вызывают reflow, работают на GPU
- Не анимировать `width`, `height`, `top`, `left`, `margin`, `padding` — вызывают reflow всей страницы
- Для плавного появления элементов — `opacity` + `transform: translateY()`, не `display` переключение
- `will-change: transform` — только если анимация тяжёлая и уже есть проблемы, не превентивно
- Читать из DOM и писать в DOM — разделять, не чередовать в цикле (вызывает thrashing)

```js
// ПЛОХО — чтение и запись чередуются, каждая итерация вызывает reflow
elements.forEach(el => {
  const h = el.offsetHeight  // читаем
  el.style.height = h + 10 + 'px'  // пишем → reflow → следующее чтение устарело
})

// ХОРОШО — сначала все читаем, потом все пишем
const heights = elements.map(el => el.offsetHeight)
elements.forEach((el, i) => { el.style.height = heights[i] + 10 + 'px' })
```

## IntersectionObserver вместо scroll-событий

- Для lazy-логики, анимаций появления, счётчиков — использовать `IntersectionObserver`
- Не вешать тяжёлую логику на `scroll` событие без `passive: true` и throttle
- В этом проекте IntersectionObserver уже реализован в `script.js` — переиспользовать

## Чеклист при добавлении изображений

- [ ] `loading="lazy"` на все изображения кроме первого экрана
- [ ] `decoding="async"` добавлен
- [ ] `width` и `height` указаны
- [ ] Формат `.webp` если возможно
- [ ] `alt` заполнен осмысленно

## Чеклист при добавлении скриптов

- [ ] Скрипт в конце `<body>` или с `defer`
- [ ] Не блокирует загрузку страницы
- [ ] Нет тяжёлых синхронных операций при старте
